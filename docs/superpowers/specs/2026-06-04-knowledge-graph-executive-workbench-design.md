# 知识图谱 Executive Workbench UI 提升设计

日期：2026-06-04

## 背景

当前应用是 React + Vite 的本地知识图谱工具，主界面由 `AppShell`、`KnowledgeBase`、`KnowledgeBaseGraphZone`、`GraphCanvas` 和 `Inspector` 组成。现有功能已经完整，包括图谱编辑、搜索、筛选、节点检查器、文件打开保存、撤销重做和项目管理页。

本次目标是使用 GSAP 做一次全面的 UI 提升，风格方向为“麦肯锡式咨询合伙人工作台 + 研究分析 cockpit”。用户已确认采用 Executive Workbench 方案，并明确范围为：重点重构知识图谱页，项目管理页只保持全局主题一致。

## 目标

- 将知识图谱页升级为专业、克制、高信息密度的分析工作台。
- 提升侧栏、顶部工具栏、画布、节点、右侧检查器、状态提示和搜索体验的视觉层级。
- 使用 GSAP 增加克制、可中断、性能友好的动效。
- 保持现有功能、数据格式和文件兼容性不变。
- 保持 macOS 和 Windows 双端可运行。
- 所有新增文本和文件优先使用 UTF-8。

## 非目标

- 不重构项目管理页，只让它继续继承全局主题变量。
- 不修改 `.graph` 文件格式。
- 不改变文件打开和保存依赖浏览器 File System Access API 的产品决策。
- 不重写 React Flow 的核心图谱交互。
- 不改变现有搜索、筛选、拖拽、编辑、撤销重做、文件状态等业务逻辑。
- 不做营销页式大面积装饰、夸张动画或循环闪烁效果。

## 用户体验设计

知识图谱页保留现有三栏工作区结构，但整体升级为 Executive Workbench：

- 左侧导航成为更稳定的工作区 rail，用深色结构色强调当前模块。
- 中间顶部工具栏改为高密度控制条，整合节点/连线数量、文件状态、搜索、筛选、文件操作和历史操作。
- 画布保持明亮背景和网格，但增加更清晰的浮层状态、筛选入口和缩放控制。
- 节点卡片提升层次、边框、阴影、选中态、搜索命中态和锁定态。
- 右侧检查器升级为分析面板，突出标题区、属性区、待办进度、标签和引用关系。
- 缺失图片、草稿保存失败、未保存状态等提示继续复用现有状态来源，但视觉上更清晰。

## 视觉系统

视觉语言采用克制的咨询工具风格：

- Executive Navy 用于结构和深色侧栏。
- Klein Blue 用于当前选择、焦点和核心操作。
- Teal 用于进度、正向状态和分析强调。
- Red 和 Amber 用于风险、错误和警告。
- 主圆角保持 8px 左右，避免过度圆润。
- 使用 1px 分隔线、轻阴影、清晰字体层级和高密度布局。
- 避免整页深色压迫感，画布和检查器保持长时间阅读友好的浅色基调。

## GSAP 动效设计

GSAP 只负责 UI 动效，不持有业务状态。

计划动效包括：

- 首屏进入：侧栏、工具栏、画布和检查器按层级淡入，并使用轻微 `y` 位移。
- 面板切换：侧栏折叠、检查器折叠、状态提示进入和离开使用 `autoAlpha`、`x`、`y`。
- 节点反馈：节点选中、搜索跳转、快速编辑进入时使用轻微 `scale` 和透明度反馈。
- 搜索结果：下拉结果进入、键盘高亮切换使用短时 transform 动效。
- 状态提示：错误、未保存、缺失图片等提示使用短促进入，不循环闪烁。

性能和无障碍约束：

- 优先动画 `opacity`、`autoAlpha`、`x`、`y`、`scale`。
- 避免动画 `width`、`height`、`top`、`left`、`margin` 等布局属性。
- 使用 `gsap.matchMedia()` 支持 `prefers-reduced-motion`。
- 在 React 中使用 scoped cleanup 模式；如引入 `@gsap/react`，优先使用 `useGSAP` 和 `scope`。

## 技术设计

实现保持现有代码边界：

- `KnowledgeBase` 继续负责状态编排。
- `KnowledgeBaseGraphZone` 继续组合工具栏、消息区和图谱画布。
- `GraphCanvas` 继续承载 React Flow 画布、节点、边、上下文菜单和画布控制。
- `KnowledgeBaseInspectorPanel` 和 `Inspector` 继续承载右侧检查器。
- 新增或调整的代码集中在知识图谱页 UI 层、样式层和小型 GSAP 动效 hook/组件层。
- 全局设计 token 放在现有 CSS 变量体系中，让项目管理页自然保持主题一致。

数据流保持不变：

- 搜索、筛选、选择、历史、文件状态、图片资源、节点/边更新仍使用现有 hooks。
- GSAP 根据 React 状态触发动画，但不直接修改业务数据。
- `.graph`、`.project` 文件读取保存流程不变。

## 错误处理与降级

- 缺失图片资源继续使用现有 `missingImageAssetCount`。
- 草稿保存失败继续使用现有 `draftSaveFailed`。
- 文件错误继续使用现有 `status.errorMessage`。
- 动效加载失败不应阻断核心操作；UI 必须在无动画情况下可用。
- 低动效模式下禁用首屏编排和复杂转场，只保留必要透明度变化。

## 测试与验证

实现时需要覆盖：

- 工具栏状态、搜索、筛选、文件操作按钮和历史按钮不回退。
- 搜索结果下拉的键盘导航、关闭、跳转仍可用。
- 检查器折叠、节点属性、待办、标签和引用区域仍可用。
- 节点选中、搜索命中、锁定、过滤置灰等状态样式完整。
- 深色/浅色主题下文字可读，按钮和面板不重叠。
- 桌面和窄屏视口下首屏不空白、不溢出、不遮挡关键操作。

完成实现后需要运行：

- `npm run typecheck`
- `npm run lint`
- 相关测试或 `npm test`
- `npm run build`
- 如架构或代码已修改，运行 `codegraph init -i` 并复扫关联项。

## 实施边界

第一阶段只交付知识图谱页 UI 与 GSAP 动效升级。项目管理页只通过共享 token 保持视觉一致，不进行组件级重构。任何会改变文件格式、核心业务逻辑、非 Chromium 兼容策略或跨平台运行方式的内容都不包含在本次实施中。
